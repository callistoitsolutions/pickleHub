How to Integrate Every Page & Template
Connecting HTML pages with Django backend
💡
Plain English: Think of Django like a smart waiter. The customer (browser) asks for a page → waiter (Django) goes to the kitchen (database) → gets the data → puts it on a plate (HTML template) → serves it back.
LayerWhat it doesTechnology
URLDecides which page to show based on web addressurls.py
ViewFetches data from DB and sends to templateviews.py
TemplateHTML page that displays the dataHTMLJinja2
ModelDatabase table structuremodels.py

Step 1 — Create the HTML template file

Put your HTML file inside templates/ folder. Example: templates/products/list.html

Step 2 — Create a View in views.py

This is a Python function that fetches data and sends it to your HTML page.

# apps/products/views.py
from django.shortcuts import render
from .models import Product

def product_list(request):
    products = Product.objects.filter(is_active=True)
    return render(request, 'products/list.html', {'products': products})

Step 3 — Register URL in urls.py

Tell Django: "when user goes to /products/ → run the product_list function"

# apps/products/urls.py
from django.urls import path
from . import views

urlpatterns = [
    path('products/', views.product_list, name='product-list'),
    path('products/<slug:slug>/', views.product_detail, name='product-detail'),
]

Step 4 — Use data inside HTML template

Use double curly braces to print data. Use {% %} for loops and conditions.

<!-- templates/products/list.html -->
{% for product in products %}
  <div class="card">
    <h3>{{ product.name }}</h3>
    <p>₹{{ product.price }}</p>
  </div>
{% endfor %}

Step 5 — Header & Footer using base template

Create one base.html with header & footer. Every other page "extends" it — so changing header in one place updates everywhere.

<!-- templates/base.html -->
{% include 'partials/header.html' %}
{% block content %}{% endblock %}
{% include 'partials/footer.html' %}

<!-- templates/products/list.html -->
{% extends 'base.html' %}
{% block content %}
  <h1>Our Products</h1> ...
{% endblock %}
✅
Key rule: Every page = 1 URL + 1 View + 1 Template. That's the Django pattern.

// Example: Add to Cart using JavaScript Fetch API
document.getElementById('add-cart-btn').addEventListener('click', function() {
  fetch('/api/cart/add/', {
    method: 'POST',

    document.getElementById('cart-count').textContent = data.cart_total; // update header
    showToast('Added to cart! 🛒');
  });
});

How to Develop Settings Functionality
Admin can change site config without touching code
💡
Plain English: Create a SiteSettings table in database with ONE row. Admin edits values from panel (logo, GST%, delivery charges, email, WhatsApp number etc). Django reads from DB — not hardcoded.

Step 1 — Create SiteSettings Model

class SiteSettings(models.Model):
  site_name = models.CharField(max_length=100)
  logo = models.ImageField(upload_to='settings/')
  gst_number = models.CharField(max_length=20)
  gst_rate = models.DecimalField(default=18)
  free_delivery_above = models.PositiveIntegerField(default=499)
  cod_charge = models.PositiveIntegerField(default=40)
  support_email = models.EmailField()
  support_phone = models.CharField(max_length=15)
  whatsapp_number = models.CharField(max_length=15)
  razorpay_key = models.CharField(max_length=100, blank=True)
  low_stock_alert = models.PositiveIntegerField(default=10)

Step 2 — Make it available on every page (context processor)

Add a context processor so every template can use {{ settings.site_name }} automatically.

# apps/core/context_processors.py
def site_settings(request):
  settings = SiteSettings.objects.first()
  return {'site_settings': settings}

Step 3 — Admin edits via Settings page

Build a settings form in admin panel. When admin saves → updates the one DB row. No code changes needed ever.

How to Communicate via Email & WhatsApp
Automated messages on order, shipment, delivery
📧
Email — Django + Gmail SMTP — Triggered automatically when order is placed, shipped, delivered.

When to trigger emails

Use Django Signals — they auto-fire when something happens (order saved, status changed).

# apps/orders/signals.py
from django.db.models.signals import post_save
from django.dispatch import receiver

@receiver(post_save, sender=Order)
def order_placed(sender, instance, created, **kwargs):
  if created:
    send_order_email.delay(instance.id) # Celery async task
    send_whatsapp.delay(instance.id)

Send Email using Django + Gmail

from django.core.mail import send_mail
from django.template.loader import render_to_string

def send_order_email(order_id):
  order = Order.objects.get(id=order_id)
  html = render_to_string('emails/order_confirm.html', {'order': order})
  send_mail('Your PickleHub order is confirmed!', '',
            'noreply@picklehub.in', [order.customer.email], html_message=html)

How to Create Discount Coupons & Offers
Database model + validation + auto-apply logic

Coupon Database Table (Model)

class Coupon(models.Model):
  TYPE = [('percent','Percentage'),('flat','Flat'),('free_del','Free Delivery')]
  code = models.CharField(max_length=20, unique=True)
  discount_type = models.CharField(max_length=10, choices=TYPE)
  value = models.DecimalField(max_digits=6, decimal_places=2)
  max_discount = models.DecimalField(null=True, blank=True) # cap for %
  min_order = models.DecimalField(default=0)
  max_uses = models.PositiveIntegerField(default=100)
  used_count = models.PositiveIntegerField(default=0)
  expiry = models.DateField()
  is_active = models.BooleanField(default=True)

Validate coupon when customer applies it

def apply_coupon(code, cart_total):
  coupon = Coupon.objects.get(code=code.upper(), is_active=True)
  if coupon.expiry < date.today(): raise Exception("Coupon expired")
  if cart_total < coupon.min_order: raise Exception("Min order not met")
  if coupon.used_count >= coupon.max_uses: raise Exception("Coupon exhausted")
  # Calculate discount
  if coupon.discount_type == 'percent':
    disc = min(cart_total * coupon.value/100, coupon.max_discount or 9999)
  else: disc = coupon.value
  return disc

Invoice & Return Policy
PDF invoice generation + return/refund workflow

Generate PDF Invoice using WeasyPrint

from weasyprint import HTML
from django.template.loader import render_to_string

def download_invoice(request, order_id):
  order = Order.objects.get(id=order_id)
  html_str = render_to_string('invoices/invoice.html', {'order': order})
  pdf = HTML(string=html_str).write_pdf()
  response = HttpResponse(pdf, content_type='application/pdf')
  response['Content-Disposition'] = f'attachment; filename=PH-{order_id}.pdf'
  return response

Return Request Model

class ReturnRequest(models.Model):
  STATUS = [('pending','Pending'),('approved','Approved'),('rejected','Rejected')]
  order = models.ForeignKey(Order, on_delete=models.CASCADE)
  reason = models.TextField()
  status = models.CharField(max_length=10, choices=STATUS, default='pending')
  refund_amount = models.DecimalField(default=0)
  created_at = models.DateTimeField(auto_now_add=True)

SEO — Search Engine Optimization
Auto-generate titles, descriptions, keywords for every product
🔍
Plain English: SEO means making Google show your products when someone searches "buy mango pickle online Nagpur". The right words in the right HTML tags = more free traffic.

Auto-generate SEO title & description in Product model

class Product(models.Model):
  name = models.CharField(max_length=200)
  seo_title = models.CharField(max_length=70, blank=True)
  seo_description = models.TextField(max_length=160, blank=True)
  primary_keyword = models.CharField(max_length=100, blank=True)
  secondary_keywords = models.CharField(max_length=300, blank=True)

  def save(self, *args, **kwargs):
    if not self.seo_title: # auto-generate if not filled
      self.seo_title = f"Buy {self.name} Online | PickleHub Nagpur"
    if not self.seo_description:
      self.seo_description = f"Order {self.name} online. Fresh homemade pickle. Free delivery above ₹499. Authentic Nagpur taste."
    super().save(*args, **kwargs)
